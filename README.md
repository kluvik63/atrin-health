## ШАГ 1. Создание базы данных Supabase (10 минут)

Supabase — это бесплатная облачная база данных с админ-панелью. Бесплатно до 50 000 активных пользователей в месяц — на старте вам этого с запасом.

1. Откройте **[supabase.com](https://supabase.com)** → нажмите **«Start your project»**
2. Зарегистрируйтесь (проще всего через GitHub или Google)
3. Нажмите **«New project»**
4. Заполните:
   - **Name:** `atrin-health` (или любое имя)
   - **Database Password:** сгенерируйте надёжный — нажмите «Generate a password». **Сохраните его** в надёжное место (понадобится для подключения извне, но не для нашего приложения)
   - **Region:** **Frankfurt (eu-central-1)** — ближе всего к России
   - **Pricing Plan:** Free
5. Нажмите **«Create new project»**
6. Подождите ~2 минуты, пока проект создаётся

## ШАГ 2. Подключение приложения к Supabase

После создания проекта:

1. В левом меню Supabase нажмите на значок **«Project Settings»** (шестерёнка внизу)
2. Выберите **«API»**
3. Вы увидите два значения:
   - **Project URL** — это `https://xxxxxxxxxxxx.supabase.co`
   - **Project API keys → anon public** — длинная строка `eyJhbG...`

4. Скопируйте оба значения в файл **`config.js`** в вашей папке:

```js
window.APP_CONFIG = {
  supabaseUrl:     'https://xxxxxxxxxxxx.supabase.co',  // ← сюда Project URL
  supabaseAnonKey: 'eyJhbG...',                          // ← сюда anon public ключ
  mode: 'cloud',
  // ... остальное оставить как есть
};
```

> ⚠ **Безопасность:** anon-ключ можно публиковать — он встроен в приложение и виден любому. Доступ к данным ограничивается на уровне базы (мы настроим в шаге 3). А вот **service_role** ключ публиковать **нельзя никогда** — это пароль администратора.

## ШАГ 3. Создание таблиц (SQL)

1. В Supabase в левом меню нажмите **«SQL Editor»** → **«+ New query»**
2. Откройте файл `supabase-schema.sql` из папки и **полностью скопируйте его содержимое**
3. Вставьте в редактор Supabase
4. Нажмите **«Run»** (или Ctrl+Enter)
5. Должно появиться: **«Success. No rows returned»**

Готово — таблицы созданы.

Проверим: в левом меню нажмите **«Table Editor»** — должно появиться три таблицы:
- `users` (клиенты)
- `entries` (ежедневные записи)
- `events` (аналитика)

## ШАГ 4. Деплой на Vercel

Так же, как в v1:

1. Заходите на **[vercel.com](https://vercel.com)** → **Add New → Project**
2. Перетаскиваете папку `atrin-pwa-v2` в окно (или используете CLI)
3. Vercel автоматически разворачивает приложение
4. Через минуту получаете адрес типа `atrin-health.vercel.app`

**Важно:** убедитесь, что вы внесли свои ключи в `config.js` **до** деплоя. Если внесёте после — перезапустите деплой.

## Как смотреть статистику и базу клиентов

### Базовый просмотр (без SQL)

1. Supabase → **Table Editor**
2. Выберите таблицу `users` — увидите всех клиентов с телефонами и городами
3. `entries` — все ежедневные записи
4. Можно **фильтровать, сортировать, экспортировать в CSV** прямо из интерфейса

### Готовые отчёты (SQL)

В файле `analytics-queries.sql` я собрал готовые запросы для отчётности:

- Сколько всего пользователей и записей
- Воронка активации (% дошли до 7-го, 30-го дня)
- Регистрации по дням/неделям
- Самые активные пользователи
- «Спящие» клиенты (не заходили 14+ дней)
- Распределение по городам
- Какие добавки и привычки чаще выбирают
- Когортный анализ удержания

Как использовать:
1. Supabase → **SQL Editor → + New query**
2. Скопируйте нужный запрос из `analytics-queries.sql`
3. Нажмите **«Run»**
4. Результат можно скачать в CSV (кнопка экспорта вверху)

### Пример: список ваших клиентов

```sql
select
  phone, name, city,
  to_char(created_at, 'DD.MM.YYYY') as registered,
  to_char(last_active_at, 'DD.MM.YYYY') as last_seen,
  days_tracked
from user_stats
order by created_at desc;
```

Запустили — увидели всю клиентскую базу.

### Привязка к покупкам в магазине

После того как клиент купил у вас витамины, отметьте его в базе:

```sql
update users
set is_customer = true,
    customer_since = current_date,
    notes = 'Купил D3+K2 22.05.2026'
where phone = '+79991234567';
```

После этого в выгрузке клиентов вы будете видеть, кто из приложения уже стал покупателем — и можно делать им персональные предложения.

## Управление контентом приложения

Тексты, список добавок и привычек — **всё в файле `config.js`**:

```js
welcomeTitle:    'Добро пожаловать',
welcomeSubtitle: 'Ваш персональный дневник здоровья',
welcomeBenefits: [
  'Сохраним ваши данные между устройствами',
  'Личные рекомендации по добавкам',
  'История не пропадёт при смене телефона',
],

supplements: [
  { id: 'd3k2',  label: 'D3 + K2' },
  { id: 'mg',    label: 'Магний' },
  { id: 'omega', label: 'Омега-3' },
  // ← добавьте сюда свои продукты ATRIN LAB
],
habits: [
  { id: 'water', label: '8 стаканов воды' },
  // ← добавьте свои привычки
],
```

**Чтобы поменять текст или добавить продукт:**

1. Откройте `config.js` в любом текстовом редакторе
2. Поменяйте нужную строку
3. Сохраните файл
4. Закиньте обновлённый `config.js` на Vercel:
   - Самое простое: на Vercel → ваш проект → **Deployments → ... → Redeploy** (с заменённым файлом)
   - Через CLI: `vercel --prod` в папке

> ⚠ **Важно про `id`:** Не меняйте `id` существующих продуктов и привычек (`'d3k2'`, `'water'` и т.п.). Это уникальные идентификаторы, по которым данные хранятся. Менять `label` — можно сколько угодно. Добавлять новые — можно.

### Пример: добавить новую добавку

```js
supplements: [
  { id: 'd3k2',  label: 'D3 + K2' },
  { id: 'mg',    label: 'Магний' },
  { id: 'omega', label: 'Омега-3' },
  { id: 'collagen', label: 'Коллаген' },   // ← новое
  { id: 'probio',   label: 'Пробиотики' }, // ← новое
],
```

После деплоя у пользователей в дневнике появятся два новых чекбокса.

## Что дальше — следующие шаги

Когда наберётся 200+ активных пользователей, имеет смысл подключить:

### 1. SMS-верификация телефонов
Чтобы убедиться, что номера настоящие.
- **Цена:** ~3₽ за SMS × количество регистраций
- **Сервисы:** SMSC.ru, SMS.ru, Smsaero (для России)
- **Сложность:** 2-3 дня работы программиста

### 2. Push-уведомления
Анонсы акций, напоминания о покупке.
- **OneSignal** — бесплатно до 10 000 подписчиков
- **Сложность:** 1-2 дня работы

### 3. Аналитика поведения
Подробные метрики, воронки, A/B-тесты.
- **PostHog** или **Mixpanel** — бесплатные тарифы есть
- **Сложность:** 1 день

### 4. Интеграция с кассой / 1С
Автоматические напоминания через 30 дней после покупки.
- **Сложность:** зависит от кассы, 2-6 недель
- Для МойСклад / Контур / iiko есть готовые API

### 5. Магазин приложений
Приложение в App Store и Google Play.
- **Capacitor** оборачивает PWA в нативное приложение
- **Цена:** $99/год Apple + $25 разово Google
- **Сложность:** 2 недели

### 6. Привязка к продажам
Отметка «купил витамины» в карточке клиента, автоматическая выгрузка из CRM.
- Через таблицу `users.is_customer` + регулярная синхронизация

## Юридическая часть (152-ФЗ)

Поскольку собираете персональные данные (телефон), технически попадаете под 152-ФЗ. Минимум для соблюдения:

1. **Согласие** — у нас в форме регистрации есть чекбокс «Согласен на обработку...»
2. **Политика конфиденциальности** — нужно создать страницу `privacy.html` с текстом и сослаться на неё в `config.js` (consentText)
3. **Регистрация оператора** (необязательно для малых обработок, но рекомендуется) — на сайте Роскомнадзора
4. **Защита данных** — Supabase шифрует данные, у них SOC 2 сертификация. Этого достаточно.

Шаблон политики можно сделать через **iubenda.com** или **termly.io** (~$10/мес). Или попросите юриста подготовить — 5-10к ₽.

## Поддержка

Если что-то не работает:
1. Откройте приложение в браузере (не как «иконку» на телефоне)
2. Нажмите F12 → Console
3. Смотрите ошибки. Самое частое: неправильные ключи в `config.js` — приложение жалуется на «Invalid API key» или похожее.

Если в `config.js` ключи не вставлены или содержат `ВАШ-ПРОЕКТ` — приложение работает в локальном режиме без облака, без регистрации. Чтобы включить cloud-режим, заполните оба ключа.
